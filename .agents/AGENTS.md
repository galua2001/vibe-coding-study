# 프로젝트 규칙 (Project Rules)

## 1. 한국어 소통 규칙
- 모든 대화와 답변은 항상 친절한 한국어(한글)로 진행해 주세요.
- 코드 작성 시 주석과 문서 설명 역시 한글로 작성해 주세요.

## 2. 하위 에이전트 팀 협업 규칙
- 새로운 코딩 실습이나 개발 프로젝트 지시를 받으면, 단독으로 처리하지 마세요.
- 자동으로 설계, 개발(코더), 검증(테스터) 등 역할이 나뉜 하위 에이전트(Subagent)들을 정의(define_subagent)하고 이들과 협업(invoke_subagent)하여 결과물을 완성해 주세요.
- 하위 에이전트들에게도 '한국어 사용 규칙'을 동일하게 전달해 주세요.

## 3. Git 버전 관리 규칙
- 새로운 프로젝트 폴더에서 작업을 시작할 때, 기존에 Git 저장소가 설정되어 있지 않다면 즉시 `git init`을 실행하여 저장소를 초기화해 주세요.
- 코드 작성 및 주요 작업 단계가 완료될 때마다 한글로 커밋 메시지를 작성하여 Git 커밋을 남겨주세요.

## 4. 가상환경 및 패키지 관리 규칙
- 파이썬 프로젝트나 실습 폴더를 시작할 때, 컴퓨터에 `uv`가 설치되어 있다면 `uv venv`를 실행하여 항상 가상환경(`.venv`)을 먼저 생성해 주세요.
- 패키지를 설치하거나 코드를 실행할 때는 반드시 이 가상환경(`.venv`)을 활성화한 후, `uv pip install` 등을 사용하여 프로젝트별로 독립적으로 관리해 주세요.

## 5. 코딩 스타일 및 설명 규칙
- 코드 작성 시 항상 변수 이름 옆에 간단한 설명 주석을 꼭 달아주세요.
- 새로운 외부 라이브러리를 설치할 때는 먼저 사용자에게 물어보고 가상환경에 설치해 주세요.
- 에러가 나면 왜 에러가 났는지 초보자의 눈높이에 맞추어 먼저 3줄 요약 설명해 주세요.
